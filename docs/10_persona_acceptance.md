# 10 — persona_acceptance (다중 페르소나 병렬 수용 테스트)

## 목적
설치·검증을 기계 게이트(doctor C1~C9, 모듈 G)만으로 끝내지 않는다. **여러 페르소나 에이전트를 병렬로**
돌려 "정말 설치가 부드러운가 · 답이 쓸 만한가 · 예쁜가 · HW 설계에 가치가 있는가"를 사람 관점으로 판정한다.
**개발 전용**(대원칙 5) — 배포 skill에 포함되지 않는다.

## 선행 / 산출물
- 선행: 08(e2e 동작), 09(skill·doctor).
- 산출물: `tests/acceptance/` — 페르소나 러너 + 각 페르소나 리포트 + 종합 판정(synthesizer).

## 페르소나 (병렬 실행, 각자 독립 리포트)

| 페르소나 | 역할/질문 | 무엇을 보나 | 통과 신호 |
|----------|-----------|------------|-----------|
| **설치자(Installer)** | INSTALL을 위→아래로 그대로 따라간다 | 막힘·모호한 단계·침묵 실패 | 한 명령+한 검증으로 끝까지 진행 |
| **검증자(Verifier)** | doctor·모듈 게이트·데이터 무결성 | C1~C9, 인덱스 재로드, 증분 add/remove | 모든 게이트 통과, 실패 시 정확 진단 |
| **사용자(User)** | 실제 `/rag` 질의(텍스트·구조·이미지) | 답의 정확성·출처·이미지 해석 | 3유형 모두 쓸 만한 답 |
| **사용자2(User2)** | "처음 쓰는 사람"으로 **설치~사용 전반이 쉽고 직관적인가** | 온보딩 마찰, 단계 모호성, 에러가 사람 말인가, `/rag` 한 줄로 바로 쓰이나 | 문서만 보고 30분 내 성공, 막힘 0 |
| **리더(Leader)** | "이게 정말 HW 설계에 가치가 있나?" | 구조 질의·Figure·멀티모달이 ARM 매뉴얼류에 실제 유용한가 | 가치 근거 + 한계 명시 |
| **UI 디자이너(Designer)** | 예쁜가/읽기 좋은가 | README·CLI 출력·진행률·에러 메시지의 가독성·미감 | 깔끔·일관·과하지 않음 |

> 페르소나는 확장 가능(예: 이식자=사내 파서로 교체해보기, 회의론자=일부러 깨뜨리기). 위 6개가 기본.

## 설계 결정
- **병렬 실행**: 각 페르소나 = 독립 에이전트. 서로의 결과를 모른 채 동시에 돌려 편향·사각 줄임.
- **각자 구조화 리포트**: `{persona, verdict: pass|warn|fail, findings:[...], evidence, fix_suggestion}`.
- **종합 판정(synthesizer)**: 리포트 병합 → 중복 제거 → 심각도 랭킹 → **수용/조건부/거부** 1줄 결론 + 액션 목록.
  결론에는 **"설치~사용 전반 직관성" 점수/코멘트를 반드시 포함**한다(쉬움·막힘 여부가 1급 판정 축).
- **실패는 모듈 책임으로 분류**(08과 동일): 어느 단계(파서/인덱스/검색/agent/skill/문서)가 원인인지.
- 페르소나 평가는 **인간 판단을 모사**(정성). 기계 게이트(G)와 **상호 보완**이지 대체가 아니다.
- (상위 워크플로 규칙: 멀티 에이전트 병렬, 놀리지 말 것 — 이 단계가 그 적용처.)

## 단독 실행
```bash
python -m tests.acceptance.run --personas all --config config.yaml   # 6 페르소나 병렬 → 종합 판정
python -m tests.acceptance.run --personas installer,user2            # 일부만
```

## 검증 — 작업 중 (Inline + 게이트)
- Inline: 페르소나 1개(설치자) 단독 실행으로 리포트 스키마가 나오는지 스모크.
- **게이트 G**: 6 페르소나 병렬 완주, 각자 리포트 산출, synthesizer가 단일 판정+액션 목록 생성.
  **거부(fail)면 그 액션이 어느 모듈로 가는지 분류**되어 있어야 한다. → 회귀 고정(결정적 모킹으로 한 벌).

## 검증 — 사용 중 (런타임 가드)
- 한 페르소나가 죽어도 나머지는 진행(부분 결과 + 누가 죽었는지 명시). 침묵 실패 금지.
- 종합 판정은 근거(evidence) 없는 결론을 내지 않는다.
