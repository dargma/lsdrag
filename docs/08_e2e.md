# 08 — e2e (전체 흐름 검증)

## 목적
전체 흐름(파싱→인덱스→agent→답변)을 골든 QA 셋으로 검증.

## 선행 / 산출물
- 선행: 02~07 전부.
- 산출물: `tests/e2e/` — 골든셋 + 채점 스크립트.

## 설계 결정
- 골든셋은 ARM v8-A 예시 조각 기반, 3유형 포함:
  - **텍스트 질의**(개념·정의)
  - **구조 질의**(특정 레지스터·명령어가 몇 페이지·어느 표/figure)
  - **이미지 필요 질의**(비트필드 다이어그램 해석)
- (예시는 1조각이지만) 멀티 문서 확장 시 doc 경계를 넘는 질의도 포함 가능하게 설계.
- 채점은 정답 매칭(정확/부분)으로 단순 시작. 임계값을 게이트로.
- 실패 케이스는 **어느 모듈 책임인지 분류 가능하게 로깅**(파서/인덱스/검색/agent).

## 단독 실행
```bash
python -m tests.e2e.run --config config.yaml          # 골든셋 채점
```

## 검증 — 작업 중 (Inline + 게이트)
- Inline: 골든셋 1건으로 파이프라인 끝까지 도는지 스모크.
- **게이트 G**: 골든셋 전체 정답 매칭율이 사전 합의 임계 이상. 3유형 각 최소 1건 통과. → 회귀 고정.

## 검증 — 사용 중 (런타임 가드)
- 실패 시 모듈 책임 분류 로그(파서/인덱스/검색/agent)로 어디서 깨졌는지 즉시 드러냄. 침묵 실패 금지.
